# Web App Specification: Ontario Trans Name Trends Explorer

## Overview

A single-page, static-file web application that makes the Ontario Gazette / baby name research dataset interactive and explorable. Runs locally via `python -m http.server` or any static host (GitHub Pages, Netlify). No backend required.

**Entry point:** `app/index.html`
**Data:** Pre-generated JSON files in `app/data/` (produced by `scripts/export_web_data.py`)
**Charts:** Plotly.js via CDN (consistent with existing Python analysis)
**Style:** Vanilla CSS — clean data dashboard aesthetic, dark sidebar nav, neutral content area

---

## Tech Stack

| Concern | Choice | Reason |
|---|---|---|
| HTML | Plain HTML5 | No build step, easy deployment |
| CSS | Vanilla CSS (custom) | No dependencies; dashboard aesthetic |
| JavaScript | Vanilla ES modules | No build step; modern browsers support natively |

### `baby_names.json`
Baby name frequency by year, compact format. Includes names with ≥3 occurrences in any year.
```json
{
  "ALICE": [[1950, 0.00421], [1951, 0.00398], ...],
  "OLIVER": [[1990, 0.00012], ..., [2023, 0.00891]],
  ...
}
```
Approximate size: ~5–8 MB (covers ~15,000 unique names × ~50 years average).

### `name_changes.json`
All 72,013 scored records. **Privacy: no old names, no last names.**
```json
[
  {"n": "JESSICA",  "y": 2009, "d": 0.92, "cs": -0.31},
  {"n": "OLIVER",   "y": 2013, "d": -0.88, "cs": 0.33},
  ...
]
```
Fields: `n`=new_first_name, `y`=year, `d`=gender_delta, `cs`=curve_score (null if outside window).
Approximate size: ~4 MB.

### `correlation.json`
Cross-correlation results for names with sufficient data.
```json
[
  {"name": "JESSICA", "lag": -2, "r": 0.71, "p": 0.003, "is_trans": true, "dir": "MTF"},
  {"name": "OLIVER",  "lag": 3,  "r": 0.83, "p": 0.001, "is_trans": true, "dir": "FTM"},
  ...
]
```
~1,272 rows. Small file (~80 KB).

### TICKET-03: Name Explorer Tab
**File:** `js/explorer.js`

**Layout:**
```
┌─────────────────────────────────────────────────────┐
│ Search: [________________] [+ Compare]              │
├─────────────────────────────────────────────────────┤
│  Baby name frequency: ALICE (1913–2023)             │
│  ┌───────────────────────────────────────────────┐  │
│  │  Line chart with vertical gazette markers     │  │
│  │  Markers: red=MTF, blue=FTM, grey=unclear     │  │
│  └───────────────────────────────────────────────┘  │
├──────────────┬──────────────────────────────────────┤
│ Curve score  │ Gazette appearances                  │
│ ─0.31 ██▓   │ 2003 — FTM (δ=−0.92)                │
│              │ 2007 — MTF (δ=+0.88)                │
│ Lag: r=0.71  │ 2011 — unclear (δ=+0.12)            │
│ p=0.003      │ [48 total appearances]               │
└──────────────┴──────────────────────────────────────┘
```

**Behaviour:**
1. Search box with typeahead from `baby_names.json` keys (debounced, 200ms)
2. On name select: call `renderExplorer(name)`
3. `renderExplorer(name)`:
   a. Build time-series from `baby_names.json[name]`
   b. Look up all records for this name in `name_changes.json` (pre-indexed by name)
   c. Render Plotly line chart with vertical dashed lines for each gazette year
      - Line colour: red if MTF, blue if FTM, grey if |δ| < 0.7
      - Hover on line: shows year + direction
   d. Populate stats panel: mean curve score (coloured badge), lag/r/p (if in correlation.json)
   e. Populate gazette list: year + direction (no old names)
4. Compare mode: clicking "+ Compare" shows a second search box; second name overlaid on same chart in dashed line style; own stats shown in side-by-side panels below

**Edge cases:**
- Name not in baby_names.json: show "No baby name data for [name]" in chart area; still show gazette appearances if any
- Name not in name_changes.json: show "No gazette appearances" in list
- Curve score null: show "No score (outside 2000–2018 window)"

### TICKET-05: Visualizations Tab
**File:** logic in `main.js`, chart functions in `charts.js`

Sub-tabs (horizontal pill nav within the tab):

```
[Lag Explorer] [Curve Scores] [Dose Response] [MTF vs FTM] [Pre/Post 2006] [Top Names]
```

#### 5a. Lag Explorer
Port of `viz/lag_explorer.html` logic into the web app.
- Scatter plot: x=best_lag, y=name (sorted by lag), coloured by is_trans
- Slider: filter by minimum |delta| threshold
- Search box: type a name to highlight it (opacity 1.0, others fade)
- Point size: large if p<0.05, small if not
- Click point → navigate to Name Explorer for that name

#### 5b. Curve Score Dashboard
3-panel layout (vertical stack):
- Panel 1: Violin plots for MTF, FTM, non-trans curve score distributions
- Panel 2: Mean curve score by year (2000–2018), separate lines for MTF/FTM/all-trans, with OLS trend lines and 95% CI band
- Panel 3: Scatter of curve_score vs gender_delta, coloured by 5-year chunk

Data source: aggregate from `name_changes.json` (computed client-side on load).

#### 5c. Dose Response
- X: |gender_delta| threshold (0.1 to 1.0 in 0.1 steps)
- Y: mean best lag for names passing that threshold filter
- Single line with dots; trend line; annotation: "At threshold X, mean lag = Y years"
- Data source: `correlation.json` — computed client-side by filtering on |d| threshold

#### 5d. MTF vs FTM Histogram
- Overlaid histograms of best_lag for MTF vs FTM names (from `correlation.json`)
- Legend + stats box: mean ± SD for each group, p-value from t-test (computed client-side)

#### 5e. Pre/Post 2006
- Side-by-side histograms: lag distribution for records before 2006 vs after
- Filter `name_changes.json` by year, then lookup lags from `correlation.json`

#### 5f. Top Names
- Two horizontal bar charts side-by-side: Most Ahead (top 15) and Most Behind (bottom 15)
- Data from `summary.json.top_ahead_names` and `top_behind_names`
- Click bar → Name Explorer

## Running Locally

```bash
# Generate data files first (one-time, after analysis is complete)
source .venv/Scripts/activate
python scripts/export_web_data.py \
  --raw-dir data/raw \
  --processed-dir data/processed \
  --output-dir app/data

# Serve the app
python -m http.server 8080 --directory app
# Open http://localhost:8080
```

Note: The app does **not** work as a bare `file://` URL due to CORS restrictions on `fetch()`. A local HTTP server is required. `python -m http.server` requires no installation.

## Deploying

The entire `app/` directory is a self-contained static site. To deploy:
- **GitHub Pages:** Push repo, enable Pages for `app/` folder or move `app/` to root
- **Netlify:** Drag-and-drop `app/` folder to Netlify dashboard
- **Any static host:** Upload `app/` contents to the webroot

The only requirement is that all files in `app/` are served from the same origin (so `fetch()` calls to `data/*.json` work).
